Local LLMs

April 30, 2026

Run LLMs locally for better privacy, control, and cost efficiency.

This is a brief guide to setting up a local LLM for coding tasks. At a high level, we need three components:

  • LLM Model
  • Coding Harness
  • Model Dock*

LLM Model#

Large Language Models are available on Hugging Face. You can choose a model based on factors such as size, quantization, and capabilities. This guide uses Qwen 3.6 27B.

Coding Harness#

A coding harness is a tool that uses an LLM to assist with coding tasks. OpenCode, Pi are a few open source options. Pi is a minimal terminal coding harness. Pi is also highly customizable.

Model Dock#

A Model Dock is a wrapper around inference engines that makes them easier to use. An inference engine is responsible for:

  • loading the trained model
  • running inference efficiently

For example:

  • llama.cpp — low-level, high-performance inference engine
  • Ollama
  • LM Studio

In this guide, we will use llama.cpp directly instead of a wrapper.

Steps#

  1. Install llama.cpp

    Refer to this installation guide for various options. In this blog, I will be using the steps for a macOS.

    brew install llama.cpp

  2. Start llama server llama.cpp provides both CLI and server modes. Since we want to connect it to Pi, we will run it in server mode. We also need to point it to a model from HuggingFace as mentioned before.

    llama-server -hf unsloth/Qwen3.6-27B-GGUF Replace unsloth/Qwen3.6-27B-GGUF with any other preferred model from HF. This command:

    • downloads the model (if not already present)
    • starts a local inference server

  3. Install Pi Pi has options to use package managers (npm, pnpm, bun) or via shell script.

    curl -fsSL https://pi.dev/install.sh | bash

    You may need Node.js and npm installed.

  4. Add local model to Pi Edit ~/.pi/agent/models.json

Example configuration:

{
  "providers": {
    "llama-cpp": {
      "baseUrl": "http://localhost:8080",
      "api": "openai-completions",
      "apiKey": "none",
      "models": [
        {
          "id": "Qwen3.6-27B-GGUF",
          "name": "Qwen 3.6 27B local",
          "reasoning": true,
          "contextWindow": 62000,
          "maxTokens": 32000
        }
      ]
    }
  }
}

Pi supports multiple API types. Refer to the supported APIs page for details.

  1. Run Pi

    pi This starts the coding harness and connects to your local model. You can now interact with your local LLM.

* “Model Dock” is not a standard term. I am using it to group tools like LM Studio and Ollama.
